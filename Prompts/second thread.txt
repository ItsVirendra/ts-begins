# TypeScript Concepts Covered

### Interface

```ts
interface User {
    id: number;
    name: string;
    email: string;
    role: string;
    active: boolean;
}
```

Learned:

* Interface describes data shape.
* Interface is not business logic.
* Interface belongs in `user.types.ts`.

---

### Typed Arrays

```ts
User[]
```

Examples:

```ts
let users_list: User[] = [];
let active_users: User[] = [];
```

Learned:

* `[]` alone is not enough.
* TypeScript cares about array contents.

---

### Typed Objects

```ts
const user_details: User = {
    ...
}
```

Learned:

* TypeScript validates required properties.
* Missing `active` caused a compile error.

---

### Function Return Types

Current expected signatures:

```ts
function addUser(email: string, name: string, role: string): boolean

function getUserByEmail(email: string): User | undefined

function getActiveUsers(): User[]

function updateUser(email: string, role: string, name: string): boolean

function removeUser(email: string): boolean
```

Learned:

* Return types should reflect actual behavior.
* `boolean`
* `User[]`
* `User | undefined`

---

### Union Types

```ts
User | undefined
```

Learned:

* Function may return either value.
* TypeScript forces handling both cases.

Example:

```ts
const user = getUserByEmail("missing@gmail.com");
```

This correctly causes:

```text
'user' is possibly 'undefined'
```

Safe usage:

```ts
if (user) {
    console.log(user.email);
}
```

---

### Type Narrowing

Learned through:

```ts
if (user) {
    ...
}
```

Inside the block TypeScript knows:

```ts
user
```

is not undefined.

---

### Object Shorthand

Learned:

```ts
name: name
role: role
```

can become:

```ts
name,
role,
```

when property and variable names match.

Also learned:

```ts
emailId
```

is probably a worse name than:

```ts
email
```

because it is not an ID.

---

### Export / Import

Recently discovered after VS Code refresh exposed errors:

```text
Cannot find name 'User'
Cannot find name 'users_list'
```

Understanding achieved:

```text
user.types.ts
    exports User

users.ts
    exports users_list

userManager.ts
    imports User
    imports users_list
    exports functions

index.ts
    imports functions
```

Mental model:

```text
export
    = make available

import
    = use something from another file
```
